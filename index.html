Brugada Syndrome Detection from 12-Lead ECG

A machine learning pipeline that extracts four clinically-grounded features from Lead V1 of 12-lead ECG recordings and classifies each patient as Brugada syndrome or normal sinus rhythm using a Support Vector Machine. Performance is estimated via nested 5-fold cross-validation on the Brugada-HUCA dataset (PhysioNet).

Methodology

Raw WFDB recordings from PhysioNet are processed through a signal-processing chain, features are extracted beat-by-beat, and a class-weighted SVM is evaluated via nested cross-validation.

Signal Processing

BUTTERWORTH BANDPASS FILTER

4th-order zero-phase filter via filtfilt(), passing 0.5 – 40 Hz. Removes baseline wander and high-frequency noise without phase distortion.

PAN-TOMPKINS R-PEAK DETECTION

Differentiate → square → moving-window integration (W = 150 ms) → threshold at 50% of maximum → 200 ms refractory period. Applied to filtered Lead V1.

Nested Cross-Validation

With 363 subjects, a simple hold-out split would be unreliable. Nested cross-validation separates hyperparameter tuning from performance evaluation: the inner loop optimises the SVM on training data only, while the outer loop evaluates generalisation on patients never seen during tuning.

OUTER LOOP

StratifiedKFold(n_splits=5, random_state=42). Preserves the Brugada/Normal ratio in each fold (~15 Brugada per fold). Produces the final unbiased performance estimate.

INNER LOOP

RandomizedSearchCV(n_iter=50, cv=5, random_state=42). Samples C and γ from log-uniform distributions: C ∈ [0.01, 1000], γ ∈ [0.0001, 10]. Total: 1,250 SVM fits per run.

CLASS WEIGHTING

w_c = N / (2 · N_c). w(Brugada) = 2.388, w(Normal) = 0.632. Compensates for the 1:3.8 class imbalance and penalises false negatives more heavily.

Results

Aggregate performance over all five outer folds. Confusion matrix values are accumulated out-of-fold predictions over the full patient set.
